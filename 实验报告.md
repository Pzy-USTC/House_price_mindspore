# 基于 MindSpore 的加州房价预测系统 实验报告

## 一、实验目的

1. 掌握华为 MindSpore 深度学习框架的基本使用方法
2. 理解多层感知机（MLP）与自注意力机制的结合方式
3. 完成从数据预处理、模型构建、训练评估到可视化部署的完整机器学习流程
4. 体验华为云 ModelArts 平台的训练环境及昇腾（Ascend）算力
5. 实现训练与推理分离的架构设计，验证模型的跨平台部署能力

## 二、实验环境

| 项目 | 配置 |
|------|------|
| 训练平台 | 华为云 ModelArts（Notebook 环境） |
| 深度学习框架 | MindSpore 2.4.10 |
| 硬件加速 | Ascend 910 / CPU（自动检测回退） |
| 数据集 | sklearn.datasets.fetch_california_housing |
| 编程语言 | Python 3.10 |
| 前端框架 | Streamlit 1.59.2 / Flask |
| 推理部署 | 香橙派（Orange Pi）嵌入式设备 |

## 三、数据集介绍

本实验采用加州房价数据集（California Housing Dataset），该数据集来源于 1990 年加州人口普查数据，包含 20,640 条样本，每条样本有 8 个特征和 1 个目标变量。

### 3.1 特征说明

| 特征名 | 含义 | 数据类型 |
|--------|------|----------|
| MedInc | 街区收入中位数 | 连续值 |
| HouseAge | 房屋房龄中位数 | 连续值 |
| AveRooms | 平均房间数 | 连续值 |
| AveBedrooms | 平均卧室数 | 连续值 |
| Population | 街区人口 | 连续值 |
| AveOccup | 平均入住人数 | 连续值 |
| Latitude | 纬度 | 连续值 |
| Longitude | 经度 | 连续值 |

### 3.2 目标变量

- **MedHouseVal**：街区房屋价值中位数，单位为十万美元

### 3.3 数据预处理

1. **数据集划分**：训练集 80%（16,512 条），测试集 20%（4,128 条），随机种子 42
2. **标准化**：使用 StandardScaler 对特征进行 Z-score 标准化，消除量纲差异
3. **Scaler 参数保存**：将 mean 和 scale 保存为 JSON 格式，供推理端复用

## 四、模型设计

### 4.1 模型架构：MLPAttention

本实验设计了一种带自注意力机制的多层感知机（MLPAttention），整体结构分为三个模块：

```
输入 (batch, 8)
    │
    ▼
┌─────────────────────────────┐
│         编码器 Encoder        │
│  Dense(8→128) → ReLU         │
│  → Dropout(0.2)              │
│  → Dense(128→128) → ReLU     │
└─────────────┬───────────────┘
              │ (batch, 128)
              ▼
─────────────────────────────┐
│      注意力模块 Attention      │
│  Q = Dense(encoded)          │
│  K = Dense(encoded)          │
│  V = Dense(encoded)          │
│  attn = Sigmoid(Q*K) * V     │
└─────────────┬───────────────┘
              │ (batch, 128)
              ▼
┌─────────────────────────────┐
│         融合层 Fusion         │
│  concat[encoded, attn]       │
│  → Dense(256→128) → ReLU     │
│  → Dropout(0.2)              │
│  → Dense(128→1)              │
└─────────────┬───────────────┘
              │
              ▼
         输出 (batch,)
```

### 4.2 关键设计说明

**编码器**：两层全连接网络将 8 维输入映射到 128 维隐藏空间，ReLU 激活引入非线性，Dropout 防止过拟合。

**注意力模块**：采用单 Token 简化版自注意力机制。对编码特征分别做 Q/K/V 线性投影，通过 Sigmoid 门控 `σ(Q⊙K)V` 实现特征间的自适应加权，增强重要特征的表达能力。

**融合层**：将原始编码特征与注意力输出拼接（256 维），再经两层全连接网络回归到 1 维输出。

### 4.3 MindSpore 实现要点

- 使用 `nn.Cell` 定义网络（对应 PyTorch 的 `nn.Module`）
- 前向传播在 `construct` 方法中实现（对应 `forward`）
- 使用 `nn.Dense` 定义全连接层（对应 `nn.Linear`）
- 权重导出为 numpy JSON 格式，实现与 MindSpore 解耦的纯 numpy 推理

## 五、训练过程

### 5.1 训练配置

| 超参数 | 值 |
|--------|-----|
| 优化器 | Adam |
| 学习率 | 0.001 |
| 批大小 | 32 |
| 训练轮数 | 200 |
| Dropout 概率 | 0.2 |
| 隐藏层维度 | 128 |

| 随机种子 | 42 |

### 5.2 训练平台

训练在华为云 ModelArts Notebook 上完成，脚本自动检测设备类型：
- 优先使用 Ascend 910 NPU 加速
- 检测不到 Ascend 时自动回退 CPU

### 5.3 训练回调

- **LossMonitor**：实时监控训练损失
- **TimeMonitor**：记录每步耗时
- **ModelCheckpoint**：每 epoch 保存模型检查点，最多保留 5 个

## 六、实验结果

### 6.1 评估指标

在测试集（4,128 条样本）上的评估结果：

| 指标 | 值 | 含义 |
|------|-----|------|
| MSE | 0.2776 | 均方误差，预测值与真实值差值的平方均值 |
| RMSE | 0.5269 | 均方根误差，与目标变量同量纲（十万美元） |
| MAE | 0.3426 | 平均绝对误差，平均预测偏差约 3.4 万美元 |

### 6.2 结果分析

- RMSE = 0.5269（十万美元）≈ 5.27 万美元，说明模型预测值平均偏离真实值约 5 万美元
- MAE = 0.3426（十万美元）≈ 3.43 万美元，中位数偏差更小
- 加州房价数据本身存在较大方差（房价范围 0.15~5.0 十万美元），该误差水平在可接受范围内

### 6.3 特征重要性分析

通过扰动法（Permutation Importance）计算各特征对预测结果的影响：

1. 对测试集样本做基准预测
2. 逐一打乱每个特征的值，重新预测
3. 计算预测变化的平均绝对值作为特征重要性

实验结果表明 **MedInc（收入中位数）** 是最重要的预测特征，这与经济学直觉一致——收入水平是决定房价的核心因素。其次是地理位置（Latitude/Longitude）和房屋属性（AveRooms/HouseAge）。

### 6.4 数据集分析

- **房价分布**：呈右偏分布，大部分街区房价集中在 1~3 十万美元区间
- **特征相关性**：MedInc 与房价呈强正相关（r ≈ 0.7），Latitude 与房价呈中等负相关

## 七、系统部署

### 7.1 训练-推理分离架构

本实验采用训练与推理分离的设计：

| 阶段 | 环境 | 依赖 |
|------|------|------|
| 训练 | ModelArts（Ascend/CPU） | MindSpore + 全部依赖 |
| 推理 | 任意 Python 环境 | 仅 numpy（纯 numpy 实现） |

训练完成后导出三个 JSON 文件：
- `weights.json`：模型权重（约 2.1 MB）
- `scaler_params.json`：标准化参数
- `metrics.json`：评估指标

推理端无需安装 MindSpore，用纯 numpy 复现前向传播，实现跨平台部署。

### 7.2 可视化前端

提供两种 Web 界面：

1. **Streamlit 完整版**（`app.py`）：特征滑块、预测结果、评估指标、特征重要性图表、数据集概览
2. **Flask 轻量版**（`web_server.py`）：适合香橙派等嵌入式设备，资源占用低

### 7.3 香橙派部署

将模型文件传输到香橙派，安装 numpy 和 flask 后运行 `web_server.py`，即可通过局域网浏览器访问推理服务。模型推理耗时仅数毫秒，完全满足实时交互需求。

## 八、实验总结

### 8.1 主要成果

1. 成功在华为云 ModelArts 平台完成模型训练，验证了 MindSpore 框架的可用性
2. 设计了 MLPAttention 模型，将自注意力机制引入房价预测任务
3. 实现了训练-推理分离架构，推理端纯 numpy 实现，不依赖 MindSpore
4. 完成从云端训练到边缘设备（香橙派）部署的完整链路

### 8.2 技术亮点

- **MindSpore 原生开发**：使用 nn.Cell、construct、nn.Dense 等 MindSpore 特有 API
- **Ascend 自动检测**：训练脚本自动适配 Ascend NPU 或 CPU 环境
- **纯 numpy 推理**：权重以 JSON 格式导出，推理代码与框架解耦
- **跨平台部署**：同一模型可在云端（ModelArts）、PC（Streamlit）、嵌入式设备（香橙派）上运行

### 8.3 改进方向

1. 增加更多网络结构对比实验（如纯 MLP、CNN、Transformer）
2. 引入学习率调度策略和早停机制
3. 使用交叉验证评估模型稳定性
4. 优化香橙派部署，加入模型量化以进一步降低推理延迟
